## 2. Architecture — SPA Router

### 2.1 Hash-based routing

Single `index.html` stays. A lightweight `js/router.js` module listens to `hashchange` and `popstate`.

**Routes:**

| Hash | Page | Content |
|---|---|---|
| `#/map` | Map | Full-screen map + overlays + bottom sheet |
| `#/plan` | Plan | Route builder + budget + prep |
| `#/fun` | Fun | Bingo + Daily Dare + FutureMe + Soundtrack |
| `#/guide` | Guide | Country/city guides + emergency + inclusion |
| `#/more` | More | AI assistant, Wrapped, settings, Turkish bonus, comparison |

**Nested routes:** `#/guide/DE`, `#/fun/bingo`, `#/more/wrapped` open the parent page and scroll/expand to the target section.

**Default:** `#/map` on first visit. Share URLs (`#route=...`) hydrate the route and land on `#/plan`.

### 2.2 Router module (`js/router.js`)

```
- parseHash(hash) → { page, sub, params }
- navigate(page, sub?) → history.pushState + render
- onRouteChange(callback) — subscribe
- getCurrentRoute() → { page, sub }
```

Each page is a lazy-loaded ES module (`js/pages/map.js`, `js/pages/plan.js`, etc.) with `mount(container)` / `unmount()` lifecycle. First import triggers load; subsequent navigations reuse the cached module.

### 2.3 Transitions

Page transitions use a simple **fade** (200ms, `cubic-bezier(0.4, 0, 0.2, 1)`). The outgoing page gets `opacity: 0`, then the incoming page renders and fades to `opacity: 1`. No slide animations — speed and simplicity for mobile.

## 4. Page Layouts

### 4.1 Map Page (`#/map`)

**Full-screen Leaflet map** — no side panel. All information comes via overlays and bottom sheet.

**Overlay elements:**
- **Top-left:** Mini summary card — route country count, total days, estimated budget. Tappable → navigates to `#/plan`.
- **Bottom-left:** Horizontal scrollable chip strip — each stop as `flag + city name` chip. Tap → map zooms to that stop.
- **Top-right:** Layer toggle buttons (Normal / Rainbow / Accessibility) + filter popover toggle.
- **Bottom-right:** Zoom controls + "Fit route" button.

**Route visualization:**
- Animated dashed polyline: blue-to-gold gradient (`--eu-blue` → `--eu-gold`), `dashArray: "8 12"`, CSS animation for dash flow.
- Numbered circle markers at each stop (1, 2, 3...) with city label below.
- Stop popup on tap: city name, country flag, nights count, transport mode badge, "Open guide" shortcut.

**Country tap → Bottom sheet:**
Slides up from bottom with drag handle. Three states:
- **Peek (30vh):** Country name + flag + scores row + "Add to route" CTA.
- **Half (50vh):** Extends to show highlights, cost-per-day, description.
- **Full (85vh):** Complete country detail (facts grid, score bars, all highlights, "Compare" button).

**Bottom sheet implementation:**
- CSS `transform: translateY()` with `cubic-bezier(0.32, 0.72, 0, 1)` for spring feel.
- Touch drag via pointer events; velocity-based snap to nearest state.
- Backdrop: semi-transparent overlay, optional `backdrop-filter: blur(4px)`.
- Dark mode: `--surface-elevated` token (slightly lighter than background).
- `will-change: transform` for GPU compositing.

### 4.2 Plan Page (`#/plan`)

**Single-column vertical scroll** with three sections flowing naturally:

**Section 1 — Route Builder:**
- 7-day progress bar (visual, shows used/remaining travel days).
- 4 seat-credit indicators (filled/empty circles).
- Drag-drop stop cards: flag, city, nights (±1 stepper), reservation warning badge.
- "Add stop" floating button at bottom of list.
- Template gallery: shown as hero when route is empty; collapsed "Browse templates" link when route has stops.
- Reservation warnings panel (collapsible, shows mandatory reservations on current route).

**Section 2 — Budget Dashboard:**
- Per-person daily cost card (big number + breakdown bar: accommodation/food/activities/transport).
- Group total card (group size × per-person).
- Mini stepper/chip controls: group size, travel style, accommodation type, food level.
- CO2 savings card (rail vs flight, green badge if ≥75% saved).

**Section 3 — Prep & Departure:**
- Countdown timer (date picker + live days remaining).
- Pre-departure checklist (10 items, progress pill, strike-through).
- Smart packing list (route-adaptive: beach items, nordic items, etc.).
- Turkish bonus cards (visa checklist, Sofia Express, TL tips, consulate reminder) — shown when applicable.

**Desktop (768px+):** 2-column layout — Route Builder on left, Budget + Prep on right.

### 4.3 Fun Page (`#/fun`)

**Visual card grid** — 2×2 on tablet+, single column on mobile.

| Card | Icon | Color accent | Content on expand |

### 4.4 Guide Page (`#/guide`)

**Sticky footer:** Emergency button (112 + local numbers + phrase pack + "show on phone" modal).

**Fewer Opportunities mode:** When active, budget sections highlight cheapest options, free activities are starred, and a "budget tips" callout appears in each country.

## 5. New & Deferred Features

### 5.1 Night Arrival Shield

**Purpose:** Filter/warn about stops where the train arrives after 22:00.

**Implementation:**
- New function in `js/features/night-shield.js`: `checkNightArrivals(route)` returns stops with estimated arrival > 22:00 based on `trains.json` duration data.
- Route Builder shows an orange "Late arrival" badge on affected stops.
- Filter toggle in the Map page filter popover: "Hide late-arrival routes".

### 5.2 Smart Packing Assistant (enhanced)

**Purpose:** Upgrade the existing `prep.js` packing list into a standalone interactive feature.

**Enhancements over v1.0:**
- Weather-aware: fetch Open-Meteo 7-day forecast for each stop city, add rain gear / sunscreen / warm layers accordingly.
- Category grouping: Essentials, Clothing, Electronics, Documents, Health.
- Custom items: user can add/remove items.
- Pack progress: visual progress bar, "all packed" celebration animation.
- Persisted in `state.prep.packingDone` (already exists).

### 5.3 Multi-LLM Adapter (Gemini + OpenAI)

**Purpose:** Support Gemini and OpenAI as alternatives to Groq, reducing single-provider risk.

**Implementation:**
- `js/features/llm-adapter.js` — unified interface: `sendPrompt(messages, options) → response`.
- Provider modules: `llm-groq.js` (existing), `llm-gemini.js` (new), `llm-openai.js` (new).
- Settings (More page): provider selector dropdown + per-provider API key input.
- All providers use browser-side fetch (CORS-friendly endpoints).
- Fallback order configurable; if primary fails, try next.

### 5.4 Group Vote Module

**Purpose:** Let a group of up to 4 travelers vote on next destination.

**Implementation:**
- `js/features/group-vote.js` — creates a vote session with a shareable URL (LZ-compressed ballot state in hash).
- Each voter opens the URL, picks from the current candidate stops, submits.
- Results rendered as a simple bar chart.
- No backend — all state in the URL hash. Works for small groups (2-4 people).

### 5.5 GPS Trip Journal (Polarsteps-lite)

**Purpose:** Passive location-based travel diary.

**Implementation:**
- `js/features/journal.js` — requests geolocation permission, logs position + timestamp to IndexedDB at configurable intervals (default: every 30 min when app is open).
- Vertical timeline UI in Fun page: day markers, city name (reverse geocoded via Nominatim or matched to `cities.json`), photo slots, short text notes.
- Photos stored in IndexedDB `journalPhotos` store, EXIF-stripped via existing `utils/image.js`.
- Export: timeline as shareable image (html2canvas) or PDF section.
- Privacy: all data stays on-device. Clear button in Settings.

### 5.6 Voice Memory Capsule

**Purpose:** Record a 30-second audio memory per day.

**Implementation:**
- `js/features/voice-memory.js` — MediaRecorder API, max 30 seconds, opus/webm codec.
- Audio blobs stored in IndexedDB `voiceMemories` store, keyed by date.
- Player UI: small waveform-style bar under each journal day entry.
- Record button with visual countdown ring (30s max).
- Fallback: if MediaRecorder unavailable, show "not supported" message (no crash).

### 5.7 Bingo Photo Collage

**Purpose:** Generate a shareable collage from completed bingo challenge photos.

**Implementation:**
- Extends existing `bingo.js` — new "Generate Collage" button appears when ≥4 challenges have photos.
- Canvas-based layout: 3×3 or 4×4 grid of cropped thumbnails with challenge label overlay.
- Export as 1080×1080 PNG via canvas.toBlob().

### 5.8 Live Wikivoyage Refresh

**Purpose:** Update guide content from Wikivoyage API on demand.

**Implementation:**
- `js/features/wikivoyage-refresh.js` — fetches Wikivoyage article for a country/city via MediaWiki API (CORS-friendly).
- "Refresh" button per guide section; shows loading spinner.
- Parsed content replaces the static `guides.json` snapshot for that entry.
- Cached in localStorage with 24h TTL.
- Phase 2 adapter in `inclusion-data.js` already has the right signatures.

### 5.9 Pickpocket Heatmap

**Purpose:** Safety awareness layer showing pickpocket risk zones.

**Data source:** Curated JSON (`data/pickpocket-zones.json`) with city-level risk areas compiled from travel forums, Numbeo crime index, and tourism safety reports. ~30 major tourist cities.

**Implementation:**
- `js/features/pickpocket.js` — renders circle markers with risk-level coloring (green/yellow/red) on the map.
- Toggle in Map page layer controls.
- City-level data: risk score (1-5), known hotspot areas (text list), safety tips.
- Shown in Guide page city detail as a "Safety" callout.

## 7. Timeline/Journal UX

**Location:** Fun page card → expands to full timeline view.

**Layout:** Vertical timeline with a thin line on the left, day nodes branching right.

**Per-day node:**
- Date + city name + country flag.
- Photo grid (up to 4 photos, tap to view full).
- Short text note (max 280 chars, inline edit).
- Voice memory player (if recorded that day).

**Interaction:**
- Scroll-based navigation, newest at top.
- "Add entry" floating button for manual entries (when GPS is off).
- Auto-entries created from GPS log + route schedule.

**Export:** "Share my journey" button → html2canvas renders the timeline as a long image, or per-day cards as individual PNGs.

## 8. Bottom Sheet System

### States
| State | Height | Content visible | Trigger |
|---|---|---|---|
| Closed | 0 | Nothing | Swipe down from peek / tap backdrop |
| Peek | 30vh | Title + key action | Country tap on map |
| Half | 50vh | Extended info | Swipe up from peek |
| Full | 85vh | Complete detail | Swipe up from half / tap "expand" |

### Implementation (`js/ui/bottom-sheet.js`)
- Single reusable component: `openSheet(content, initialState)`, `closeSheet()`.
- CSS `transform: translateY()` driven by CSS custom property `--sheet-y`.
- Touch/pointer drag with velocity detection: fast swipe → snap to next state, slow drag → snap to nearest.
- Spring easing: `cubic-bezier(0.32, 0.72, 0, 1)`, 300ms duration.
- Drag handle: 40×4px rounded bar, centered at top.
- Backdrop: `rgba(0,0,0,0.3)` with optional `backdrop-filter: blur(4px)`.
- Dark mode: sheet surface uses `--surface-elevated`, handle uses `--border-subtle`.
- Keyboard: Escape closes, Tab traps focus within sheet when open.
- `will-change: transform` during animation, removed after settle.

## 10. Responsive Strategy

### Breakpoints

| Breakpoint | Target | Layout changes |
|---|---|---|
| < 768px (mobile) | 375px base | Single column, bottom nav, full-screen map, bottom sheet for details |
| 768px–1023px (tablet) | iPad | Plan page 2-col (route left, budget right), Guide 2-col card grid, Fun 2×2 grid |
| ≥ 1024px (desktop) | Laptop+ | Map 60% + content 40% split, bottom nav → left sidebar nav, bottom sheet → side panel |

### Mobile-first approach
- All CSS starts at mobile, uses `min-width` media queries to add tablet/desktop layouts.
- Touch targets: minimum 44×44px everywhere.
- Font sizes: `clamp()` for fluid typography.
- Images: lazy-loaded, `srcset` where applicable.

### Desktop adaptations
- **Map page:** Map fills 60% left, country detail in a right panel (not bottom sheet) on desktop.
- **Bottom nav → Sidebar:** 5 items stack vertically with icons + labels, fixed left, 72px wide.
- **Plan page:** Side-by-side columns at 768px+.
- **Fun/Guide/More:** Content area maxes at 800px width, centered.

## 12. Design Tokens & Visual Identity (v1.1 polish)

### Typography
- Headings: Inter or system-ui, bold 600-700.
- Body: system-ui, regular 400.
- Sizes: fluid via `clamp()` — h1: `clamp(1.5rem, 4vw, 2.25rem)`.

### Spacing
- "Breathing" layout: minimum 16px padding on cards, 24px between sections.
- No cramped grids — generous whitespace signals quality.

### Color additions
- Fun page accents: `--fun-purple: #8B5CF6`, `--fun-orange: #F97316`, `--fun-teal: #14B8A6`, `--fun-pink: #EC4899`.
- Route polyline gradient: `--eu-blue` → `--eu-gold`.
- Bottom sheet surface: `--surface-elevated` (light: #fff, dark: #1e293b).

### Animation
- Page fade: 200ms ease-out.
- Bottom sheet spring: 300ms `cubic-bezier(0.32, 0.72, 0, 1)`.
- Route polyline dash: `@keyframes dash-flow` 2s linear infinite.
- Card hover/tap: `scale(0.98)` 100ms on press, `scale(1)` on release.
- Chip strip scroll: momentum-based, CSS `scroll-snap-type: x mandatory`.

## 13. Migration Strategy (v1.0 → v1.1)

### What stays
- `state.js`, `storage.js`, `i18n/`, `utils/` — untouched core.
- All `data/*.json` files — extended, not replaced.
- All `js/features/*.js` — reused as-is, just mounted in new page contexts.
- `css/design-system.css` — extended with new tokens.

### What changes
- `index.html` — restructured: removes old tab panel, adds `<main id="page-root">` + bottom nav.
- `js/main.js` — rewritten to use router instead of tab controller.
- `css/main.css` — rewritten for new layout (full-screen pages, bottom nav, sidebar).
- `css/components.css` — extended with bottom sheet, card grid, timeline styles.
- `js/ui/panel.js` — removed (replaced by page-level modules).
- Old tab modules (`country-detail.js`, `route-builder.js`, `budget.js`, `filters-ui.js`, `comparison.js`, `prep.js`) — refactored into page sub-components under `js/pages/`.

### New files
- `js/router.js` — hash router.
- `js/ui/bottom-sheet.js` — reusable bottom sheet.
- `js/pages/map.js`, `plan.js`, `fun.js`, `guide.js`, `more.js` — page modules.
- `js/features/night-shield.js`, `llm-adapter.js`, `llm-gemini.js`, `llm-openai.js`, `group-vote.js`, `journal.js`, `voice-memory.js`, `wikivoyage-refresh.js`, `pickpocket.js` — new features.
- `js/ui/bottom-nav.js` — navigation component.
- `data/pickpocket-zones.json` — safety data.
- `docs/outreach/eacea-one-pager.html` — outreach PDF source.
- `docs/outreach/turkish-ua-email.md` — email template.
- `docs/outreach/linkedin-dg-eac.md` — LinkedIn message.

### Data preservation
- All localStorage/IndexedDB data from v1.0 remains compatible.
- `state.panelTab` becomes unused (replaced by router); ignored on load, cleaned up on first v1.1 session.
