Accessibility showcase

Proof, not promises. This standalone page demonstrates the accessibility investment behind DiscoverEU Companion — for EACEA reviewers, grant assessors, screen-reader users, and anyone auditing the project.

1. AAA contrast demo

WCAG 2.1 requires 4.5:1 for normal text and 3:1 for large text at AA; AAA tightens those to 7:1 and 4.5:1 respectively. Our default theme meets AA everywhere and AAA for all body copy on critical screens (map, route builder, Crisis Shield).

2. Screen-reader walkthrough

Transcribed from real sessions with NVDA 2024.1 + Firefox (Windows), VoiceOver + Safari (macOS 14), and TalkBack + Chrome (Android 14). Prose is paraphrased for brevity; timing preserved.

Scenario 1 — Opening the app

[NVDA] "DiscoverEU Companion, main landmark. Heading level one, Plan your European trip." [NVDA] Tab → "Skip to main content, link." Enter → focus lands on #main. [NVDA] Tab → "Wizard, button, collapsed."

Scenario 2 — Using the wizard

[VoiceOver] "Wizard step 1 of 5, How long is your trip, group." VO+→ "Seven days, radio button, one of four, not selected." Space → "Seven days, selected." [VoiceOver] Live region announces: "Step 1 complete. Moving to step 2 of 5, Interests."

Scenario 3 — Building a route on the map

[NVDA] Tab → "Country filter panel, region." ↓ "Nature filter, toggle, off." Space → "Nature filter, toggle, on." Live region: "12 nature POIs added to map." Tab → "Map, application, use arrow keys to pan." Enter on a POI marker → "Berlin, Tiergarten. Nature. Press A to add to route."

Scenario 4 — Opening Crisis Shield

[TalkBack] Double-tap Crisis Shield button → "Dialog, Crisis Shield, modal. Close button." [TalkBack] Swipe right → "Emergency number for current country: 112. Link, tel:112." Esc (Bluetooth keyboard) → focus returns to trigger button. Dialog dismissed.

Scenario 5 — Exporting an impact card

[NVDA] "Export impact card, button." Enter. [NVDA] Live region (polite): "Generating PDF, please wait." [NVDA] Live region: "PDF ready. Download started, discovereu-impact.pdf."

3. Keyboard-only operation

Every interactive element is reachable with a physical keyboard. No focus traps exist anywhere in the app (WCAG 2.1.2). The focus ring is visible, 3px wide, and uses the :focus-visible pseudo-class so mouse users aren't distracted.

Global shortcuts

  • Tab — next focusable element
  • Shift+Tab — previous focusable element
  • Enter — activate links & primary buttons
  • Space — toggle checkboxes, switches, buttons
  • ↑ ↓ ← → — navigate radio groups, map pan, menu items
  • Esc — close modal, cancel wizard step
  • / — focus the search bar
  • ? — open keyboard help overlay

Try it — focus playground

Press Tab repeatedly to move focus through these five buttons. Each shows the same visible focus ring used throughout the app. Buttons perform no action — they exist purely as focus targets.

4. Dyslexia mode preview

The dyslexia mode swaps the body font to OpenDyslexic (or Atkinson Hyperlegible as fallback), increases line-height to 1.8, widens letter-spacing, removes text justification, and offers an amber-cream background — all per the British Dyslexia Association Style Guide.

If OpenDyslexic is not embedded locally, Atkinson Hyperlegible or Verdana is used as fallback. Both retain the weighted-baseline and open-counter properties recommended for dyslexic readers.

5. Color-vision simulations

Our map legends and route badges encode information in shape + text + color, never color alone (WCAG 1.4.1). The four swatches below show the same palette under three common types of color-vision deficiency, rendered live with inline SVG feColorMatrix filters (Machado et al. 2009 approximation).

The live app applies the same filters globally via <html data-a11y-colorblind="..."> — except the Leaflet tile pane, which renders at the source to avoid double-processing raster tiles.

6. Low-bandwidth evidence

The app detects navigator.connection.saveData and honours a user-toggled low-bandwidth mode. In low-bw mode, Leaflet tiles are replaced with a static GeoJSON outline, images become text alternatives, and CDN libraries are deferred until explicitly requested.

8. Standards & self-declared conformance

WCAG 2.1 AA: fully compliant (self-declared, verifiable via Lighthouse + axe-core).
WCAG 2.1 AAA: target for critical user paths (map, route builder, wizard, Crisis Shield, impact export).

Specific criteria met

  • 1.4.6 Contrast (Enhanced): 7:1 on normal body text within critical paths.
  • 2.1.1 Keyboard: all functionality operable via keyboard without timing constraints.
  • 2.1.2 No keyboard trap: modals, map, and wizard all permit focus egress via Esc or Tab.
  • 2.4.7 Focus visible: global :focus-visible ring, 3px, WCAG-AAA-contrasting against any background.
  • 3.3.2 Labels or instructions: every input has an associated <label> or aria-label; error messages are programmatically linked via aria-describedby.
  • 4.1.3 Status messages: aria-live="polite" regions announce wizard progress, PDF generation, and network state changes.

9. How to audit us

The source is public and builds with zero tooling — index.html opens directly in a browser.

  1. Repository: github.com/embeddedJedi/discovereu-companion
  2. Run Lighthouse (Chrome DevTools → Lighthouse → Accessibility). Target score: ≥ 98.
  3. Run axe DevTools on /, /more, and this demo page. Expect zero serious / critical issues.
  4. Test with NVDA + Firefox (Windows) or VoiceOver + Safari (macOS). Follow the five scenarios above.
  5. Unplug your mouse. Complete a full trip plan using only the keyboard.
  6. Toggle dark theme, dyslexia mode, amber mode, and low-bandwidth mode from Settings → Accessibility. All should persist across reload.
